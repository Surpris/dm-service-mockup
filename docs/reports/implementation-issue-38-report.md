## 2. 詳細評価

- **Scenario 3 (グラフ操作):** `frontend/e2e/graph-scenarios.spec.ts` にて実装済み。以下の主要機能の動作検証をカバーしています。

### ❌ 未実装 (Blocked)

- **Scenario 2 (プロジェクト閲覧):** 未実装。
  - **理由:** テスト対象となるアプリケーション機能（詳細画面、遷移ロジック）が存在しないため。

## 3. Scenario 2 実装に向けた前提課題

Scenario 2「特定のプロジェクトをクリックして詳細を表示し、関連データを確認する」をテストコード化するには、まず以下のフロントエンド機能の実装が不可欠です。

| 項目                | 必要な実装内容                                                                                              | 現状 (`frontend/src`)                 |
| :------------------ | :---------------------------------------------------------------------------------------------------------- | :------------------------------------ |
| **1. 詳細ページ**   | プロジェクトIDに基づき、詳細情報・関連データセット・貢献者を表示する `ProjectDetail` コンポーネントの作成。 | 未実装                                |
| **2. ルーティング** | `App.tsx` に `/projects/:id` 等の動的ルート定義を追加。                                                     | ルート未定義                          |
| **3. 遷移導線**     | `ProjectList.tsx` (一覧画面) の行クリックやリンクボタン等で、詳細ページへ遷移するロジックの実装。           | `DataGrid` 表示のみで遷移イベントなし |

## 4. 今後の推奨アクション

1. **機能実装フェーズ:** 上記「前提課題」に挙げたプロジェクト詳細閲覧機能の実装を行う。
2. **テスト実装フェーズ:** 機能実装完了後、`project-view.spec.ts` を作成し、シナリオテストを追加する。
3. **リファクタリング:** 今後の拡張に備え、`e2e/pages/` ディレクトリを作成し、Page Object パターンを導入してテストコードの保守性を向上させる。
